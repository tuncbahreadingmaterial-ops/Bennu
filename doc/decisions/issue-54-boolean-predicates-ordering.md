# Issue #54 — Add elementwise Boolean, predicate, and numeric-ordering primitives

- **Issue:** https://github.com/tuncbahreadingmaterial-ops/Bennu/issues/54
- **File status:** Active append-only decision record

## 2026-07-23 — Extend the existing primitive pipeline with strict pointwise predicates

- **Context:** Level 3 needs Boolean conjunction/disjunction, inequality, parity, sign, and strict numeric ordering without introducing a second evaluator, emitter-only semantics, hidden short-circuit behavior, or a new lifting/resource model. The existing registry gives every primitive and scalar implementation a stable identity, resolves exact signatures before the sole `Int -> Double` conversion, and feeds one shared scalar/vector application path to the evaluator and typed C lowering. IEEE binary64 values are normalized at value-ingress boundaries, including canonical NaN and preserved signed zero.
- **Decision:** Append stable `PrimitiveId` values 5 through 13 for `and`, `or`, `not_equals`, `odd`, `even`, `is_positive`, `is_negative`, `less_than`, and `greater_than`, in that order. Append explicit scalar implementation identities for every accepted signature and register all nine as `elementwise`. Keep `and` and `or` as ordinary eager dyadic Bool kernels; define `not_equals` as the exact selected-domain complement of `equals`; define parity with signed remainder modulo two; define sign by strict comparison with zero; and compare ordering operands exactly as written. All-Int ordering remains exact signed-64 comparison. Mixed numeric arguments select the Double signature and compare after the existing strict `Int -> Double` conversion. Both sign predicates and both strict ordering predicates return false for NaN, both signed zeros are neither positive nor negative and are not strictly ordered, and infinities use ordinary IEEE ordering. Mirror the same identities and direct kernels in deterministic generated C11 while continuing to consume the common typed lowering.
- **Alternatives considered:** Encode `greater_than[a b]` by reversing a `less_than` call; implement `not_equals` by constructing and invoking `not[equals[...]]`; convert every numeric comparison to Double; add lazy/short-circuit Boolean evaluation; add a predicate-specific lifting or resource path; or duplicate overload selection in the generated runtime. Operand reversal weakens provenance and order guarantees, composed calls add hidden work and intermediate semantics, unconditional Double conversion loses exact large-Int ordering, laziness requires an expression-evaluation capability beyond pure pointwise kernels, and separate application or selection paths would drift from established errors, charging, and cleanup.
- **Rationale:** Plain descriptors, signatures, enums, and scalar switches extend the current flat data flow without allocation or object graphs. Direct kernels make operand order, type selection, and special-value behavior locally inspectable. Reusing typed lowering and `apply_primitive` preserves scalar broadcasting, equal-vector shape checks, empty-vector zero-work behavior, result allocation, one-work-unit-per-result-element charging, preflight error order, deterministic result indices, cleanup, and evaluator/generated/native agreement automatically.
- **Consequences or follow-up:** The public primitive surface grows from five to fourteen names, while the historical BENNU-SPEC-0001 Level 2 matrix remains scoped to the four elementwise primitives it normatively froze; Issue #54 has its own current-surface and shared-lifting conformance matrix. Generated C gains matching fixed enum values and predicate branches but no new headers, allocation, runtime redispatch, or process behavior. Future syntax or optimization may not introduce short-circuit or operand-reversal observations without a separate language decision. No follow-up work is required for this issue.
- **Validation/evidence:** `ISSUE54-REGISTRY`, `ISSUE54-BOOLEAN`, `ISSUE54-NOT-EQUALS`, `ISSUE54-PARITY`, `ISSUE54-SIGN`, `ISSUE54-ORDERING`, `ISSUE54-ERRORS`, and `ISSUE54-RESOURCES` cover stable identities, signatures, truth tables, complement behavior, Int64 extrema, large-Int promotion boundaries, subnormals, signed zeros, infinities, canonical raw-NaN ingress, written operand order, vectors, empty vectors, validation precedence, exact resource refusal fields, allocation failure, and cleanup. `PUBLIC-SUCCESS-CROSS-PATH-BYTES` and `PUBLIC-SUCCESS-NATIVE` run the expanded tracked corpus through the evaluator, REPL, runner, strict generated C11, and native builder and require deterministic byte-identical output. `PUBLIC-EVALUATOR-GENERATED-NATIVE-DIAGNOSTIC-EQUIVALENCE` includes Issue #54 arity, type, and dynamic-shape failures and proves exact atomic cross-path diagnostics.
- **Supersedes:** `spec/bennu-spec-0003-scalar-domain-semantics.md`, `## 16. Non-goals and deliberate boundaries`, only where it lists floating-point ordering as undefined; approximate equality, total ordering, and the other stated non-goals remain unchanged.
