# Issue #38 — Fix Node.js 20 deprecation warnings in GitHub CI

- **Issue:** https://github.com/tuncbahreadingmaterial-ops/Bennu/issues/38
- **File status:** Active append-only decision record

## 2026-07-22 — Pin reviewed Node-24-native checkout and MSVC setup actions

- **Context:** Main CI pinned `actions/checkout` v4.3.1 and `ilammy/msvc-dev-cmd` v1.13.0 by immutable commit, but both action metadata files declare `runs.using: node20`. GitHub was forcing those actions onto Node 24 and warning on every required platform. The workflow must keep its three-platform matrix, triggers, cancellation, least-privilege permissions, Release/full-CTest journey, and exact `PR Gate` behavior while retaining immutable action pins. The ilammy repository has no released or merged Node-24-native revision; its Node 24 changes remain open pull requests.
- **Decision:** Pin `actions/checkout` v5.1.0 at full SHA `fbc6f3992d24b796d5a048ff273f7fcc4a7b6c09`, and replace `ilammy/msvc-dev-cmd` with `mlocati/setup-msvc` v1.3.1 at full SHA `ade6aff3df872d66c12a63dcacdddf0041cb2693`. Both reviewed release commits declare `runs.using: node24`. Configure `mlocati/setup-msvc` with `architecture: x64`, preserving the prior x64 MSVC environment consumed by CMake and Ninja. Make both exact identities and the absence of obsolete pins, mutable tags, and the Node-version escape hatch part of the deterministic Main CI contract, including negative mutations.
- **Alternatives considered:** `actions/checkout` v6 or v7 would also use Node 24, but they add major-version behavior beyond the runtime migration; the latest v5 line is the smallest reviewed upgrade from v4. Pinning an open ilammy Node 24 pull-request head would depend on an unreleased contributor fork. `step-security/msvc-dev-cmd` v1.13.1 is Node-24-native, but that maintained fork also adds banner and subscription-check behavior unrelated to Bennu. Replacing MSVC setup with bespoke PowerShell would remove an action but make Bennu own Visual Studio discovery and environment export logic.
- **Rationale:** The selected checkout line exists specifically for the Node 24 runtime transition, while the selected MSVC action has a small compatible interface, invokes `vcvarsall.bat`, exports its environment to later steps, and passed upstream tests on `windows-2022` plus newer Windows runners at the exact release commit. Full-SHA pins keep the executable dependency immutable. Focused positive and negative contracts make runtime-regressing or mutable substitutions visible before remote CI.
- **Consequences or follow-up:** Main CI changes action owner for MSVC setup and renames the x64 input from `arch` to `architecture`; no workflow authority or trigger changes. The new action has a shorter upstream history than ilammy, so its exact release, metadata, commit checks, and behavior must be re-reviewed when updating the pin. Remote pull-request and post-merge runs remain required to prove all platforms are green and their logs contain no Node 20 warning.
- **Validation/evidence:** Upstream release metadata and exact-commit checks were reviewed for `actions/checkout` v5.1.0 and `mlocati/setup-msvc` v1.3.1. The positive contract and four negative mutations pass; actionlint 1.7.12 and PyYAML 6.0.3 parse all three workflows; a clean Release build passes 53/53 CTests; the strict warning/conversion/error/no-exceptions build passes all 51 applicable CTests; and diff hygiene passes. Warning absence remains remote-only evidence.
